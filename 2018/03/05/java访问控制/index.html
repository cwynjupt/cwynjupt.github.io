java访问控制

访问控制：

封装，将数据封装到类的内部；将算法封装到方法中
封装结果：存在但是不可见

public：任何位置可见，修饰：属性、类、方法、内部类；跨包访问类，需要使用import语句导入

protected：当前包可见，子类中可见；修饰：类的成员（只能在类体中使用），包括：属性、方法、内部类
默认的：当前包内部可见，就是没有任何修饰词；修饰:类、属性、方法、内部类等，在实际项目中很使用

private：仅仅在类内部可见；修饰：类的成员，包括：属性、方法、内部类

默认类（包内类）的访问范围：当前包内部可见，其他包不能访问
类的访问范围受限！！！

在企业项目中建议：所有类都是公用类。。封装的类 使用内部类

Java文件规则：

  1. 一个Java文件中可以有多个类，但只能有一个共有类，其他类为默认类（包中类）而且Java的文件名

    一定与共有类名一致，如果没有共有类，可以与任何一个类名一致

  2. 一般建议：一个文件一个公有类！一般不再一个文件中写多个类

私有的方法不能继承，也不能重写（重写建立在继承之上）

static：静态的
智能在类内部使用，可以修饰：属性，方法，内部类，静态代码块
静态修饰的成员：属于类的成员，是全体类实例共享的资源
静态属性：只有一份，全体实例共享类似于全局变量
静态方法：属于类的方法，使用类名直接调用，不需要创建对象，没有隐含参数this，不能访问当前对象资源
静态代码块：在类加载期间执行，只执行一次，可以用于资源加载

final：最终的，可以修饰：类，方法，变量（属性，局部变量）
final的类：不能再被继承
final的方法：不能再被重写
final的方法和类，阻止了 动态代理模式
动态代理模式：广泛的应用在：Spring Hibernate Struts2
一般不使用final类和方法
final变量：智能初始化一次，不能再被修改

常量：public static final double PI=3.14
